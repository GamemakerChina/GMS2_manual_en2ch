偏好设置 - 插件


插件偏好设置用于控制已经添加到GameMaker Studio 2 IDE的任何额外的插件。默认有一个专门的子项源代码管理插件整合GameMaker Studio 2项目与Git工具。

本节讨论要求让 Git SCM在GameMaker Studio 2上能够使用的偏好设置。有如下常规选项：

  • 自动打开源代码管理输出窗口: 勾选此选项，在某些修改时源代码管理输出窗口会自动打开。该选项默认为“开”。

  • 启用详细日志记录（Verbose Logging）：Git 启用详细日志记录输出日志。该选项默认关闭。

接下来是 用户标识 部分，其中需要填写以下字段：
  • 提交的用户名：在这里你必须输入你将做任何提交到Gi时所使用的用户名（它必须同Git仓库所指定的用户名一致）。

  • 提交的邮箱地址：与此Git身份关联的邮箱。

设置好用户身份后，你可以通过点击添加新用户/通过认证的按钮添加新的用户名/密码组合进行身份验证，这将打开以下窗口，你可以在此填写字段。

你还可以通过单击具有 SSH 标签的按钮 添加以 SSH 密钥验证身份的账户 。 这将打开以下窗口，供你填写：

使用以下这些按钮，你可以设置 合并工具 选项 和 SCM 的 差异工具 选项：

  • 合并工具位置：在此处指定 Git 合并工具的完整文件路径。

  • 合并工具选项：在此字段你可以添加任何额外的命令，在使用合并工具时都将运行。

  • 差异工具位置: 在此处指定 Git 差异工具的完整文件路径。

  • 差异工具选项: 在此字段你可以添加任何额外的命令，在使用差异工具时都将运行。

最后一个选项：

  • 自动启用源代码管理: 选中此项，将自动为所有新项目启用源代码管理。该选项默认关闭。
